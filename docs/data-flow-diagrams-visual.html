Verification Components:

  • Motion Sensor Analysis: Analyzes accelerometer/gyroscope data from evidence capture
  • Spatial-Temporal Consistency: Checks if GPS, timestamp, and location are consistent
  • Evidence Authenticity: Validates EXIF data and checks for manipulation
  • Trust Score Calculation: Aggregates individual scores
  • Classification: Categorizes as Trusted (high score), Delayed (medium), or Fake (low)

10. Data Dictionary

Data Stores

Store ID Name Description Data Elements
D1 Report Database Stores all incident reports report_id, incident_type, description, gps_coordinates, timestamp, status, trust_level
D2 Evidence Storage Stores encrypted photo/video evidence evidence_id, report_id, media_file, metadata, exif_data, encryption_key
D3 Trust Scores DB Stores trust evaluation results report_id, motion_score, consistency_score, authenticity_score, aggregate_trust_score, classification
D4 Hotspot Data Stores detected incident clusters hotspot_id, center_coordinates, boundary_polygon, incident_count, severity_level, time_range
D5 Response Log Stores police action records action_id, report_id, officer_id, action_type, timestamp, notes

Data Flows

Flow Name Source Destination Data Elements
Anonymous Incident Report Citizen Incident Report Submission incident_type, description, gps_coordinates, timestamp
Photo/Video Evidence Citizen Evidence Capture image_file, video_file, capture_timestamp
Trust Classification Verification Engine Trust Scores DB trust_level (Trusted/Delayed/Fake), scores
Hotspot Clusters Hotspot Detection Hotspot Data cluster_coordinates, incident_density, boundaries
Trusted Incidents Report Database Police Dashboard verified_reports, locations, details
Anonymized Trends Report Database Public Safety Map aggregated_statistics, generalized_locations
